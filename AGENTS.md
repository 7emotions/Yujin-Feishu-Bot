# AGENTS.md

## Use these commands
- Install deps: `pip install -r requirements.txt`
- Full tests: `python -m pytest tests/ -q`
- Focused test: `python -m pytest tests/test_chat_plugin.py -q`
- Smoke runtime entrypoint: `python -m runtime.app --dry-run`

## Source of truth
- `README.md` is current and useful; use it for the high-level platform shape, but trust code and executable checks when details conflict.
- There are no repo-local CI / lint / formatter / typecheck configs to rely on; the executable checks are the pytest suite and the two dry-run commands above.

## Architecture that matters
- Canonical implementation lives in:
  - `runtime/` — router, session store, bootstrap, app entrypoint
  - `adapters/feishu/` — Feishu auth/settings/event adaptation
  - `toolsets/feishu/` — reusable Feishu message/file/approval operations
  - `plugins/chat/` and `plugins/reimbursement/` — plugin packages
  - `domain/reimbursement/` — reimbursement parser/workflow/form shaping
- There is no live `bot/` compatibility layer anymore; `runtime.app` is the only real entrypoint.

## Routing behavior
- `runtime/router.py` is the real router.
- It ignores self-messages via `BOT_USER_ID` from the environment and ignores non-`p2p` chats.
- Routing order is: active plugin session first, then highest `match()` score.
- `MIN_PLUGIN_MATCH_SCORE` is `0.2`; chat fallback is intentionally low-confidence but still routable.
- Session context lives in `runtime/session_store.py`; `recent_messages`, `summary`, and `last_intent` are stored per user.

## Plugin-specific gotchas
- `plugins/reimbursement/plugin.py` is thin; real behavior is in `domain/reimbursement/workflow.py`.
- `plugins/chat/plugin.py` is the fallback chat capability, not the primary router.
- Chat decisions are generated by `plugins/chat/responder.py` using the same local Qwen-VL backend as reimbursement parsing.
- The chat system prompt is built from plugin metadata (`capability_description` and `tool_descriptions` on each plugin). If you add a plugin and want chat to know about it, implement those properties.
- If you change plugin internals, keep dependency injection patchable: tests rely on monkeypatching file/message/approval modules.

## Verified quirks
- `start_bot.sh` is a real launcher and simply runs `python -m runtime.app`.
- `runtime.app` currently configures proxy environment variables internally before starting the long-lived Feishu connection.
- Logging formatters now come from the repo-local `utils/` package (`from utils import ColoredFormatter` in most modules; `runtime/app.py` still imports `from utils.utils import ColoredFormatter`).
